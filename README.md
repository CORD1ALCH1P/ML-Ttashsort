# Классификация мусора для сортировки отходов

Прототип системы сортировки отходов: по фотографии определяет материал
(бумага, пластик, стекло, металл, картон или прочий мусор), выводит
уверенность модели и рекомендацию, в какой контейнер сортировать отход.
Работает на обычном компьютере без видеокарты (CPU), тестировать можно
через веб-страницу в браузере.

---

## Быстрый старт (без опыта программирования)

Понадобится только установить Python — остальное сделают команды ниже.
Выполняются они в **терминале** (на Windows это PowerShell или "Командная
строка", на Mac — приложение Terminal, на Linux — любой терминал).

### Шаг 0. Скачать проект

Если на компьютере уже установлен Git:
```bash
git clone <ссылка-на-репозиторий>
cd <название-папки-проекта>
```

Если Git не установлен — на странице репозитория на GitHub нажмите зелёную
кнопку **Code → Download ZIP**, распакуйте архив и откройте терминал в этой
папке.

### Шаг 1. Установить Python

Нужен Python версии 3.10 или новее.

- Проверить, установлен ли он: введите в терминале `python3 --version`
  (на Windows иногда просто `python --version`).
- Если команда не найдена — скачайте Python с [python.org/downloads](https://www.python.org/downloads/)
  и установите (на Windows на первом экране установщика обязательно
  поставьте галочку **"Add python.exe to PATH"**).

### Шаг 2. Создать виртуальное окружение и установить зависимости

Виртуальное окружение — это отдельная "песочница" для библиотек проекта,
чтобы они не конфликтовали с другими программами на компьютере.

**Mac / Linux:**
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

**Windows (PowerShell):**
```powershell
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Установка займёт несколько минут (скачивается библиотека PyTorch, ~200 МБ).
Если появится ошибка — скопируйте текст ошибки и обратитесь за помощью,
не пытайтесь исправлять её наугад.

> После `activate` в начале строки терминала должно появиться `(.venv)` —
> это значит, что окружение активно. Каждый раз, открывая новый терминал
> для работы с проектом, нужно заново выполнять команду `activate`
> (но не `pip install` — библиотеки уже установлены).

### Шаг 3. Запустить веб-интерфейс

**Mac / Linux:**
```bash
./scripts/run_webapp.sh
```

**Windows:**
```powershell
streamlit run webapp/app.py
```

В терминале появится сообщение вроде `Local URL: http://localhost:8501` —
браузер должен открыться автоматически. Если нет, скопируйте этот адрес и
откройте вручную в браузере.

### Шаг 4. Проверить классификацию

1. Нажмите **"Browse files"** и выберите фото мусора (можно сфотографировать
   что-то на столе телефоном и перекинуть на компьютер, или взять готовые
   тестовые фото из папки `data/split/test/<класс>/`, если она есть в проекте).
2. Система покажет: определённый класс (бумага/пластик/стекло/металл/картон/
   прочий мусор), процент уверенности и рекомендацию, в какой контейнер
   сортировать отход.
3. В левой панели можно переключить модель (архитектуру) и посмотреть,
   как разные модели классифицируют одно и то же фото.

Чтобы остановить приложение — вернитесь в терминал и нажмите `Ctrl+C`.

### Возможные проблемы

| Проблема | Решение |
|---|---|
| `command not found: python3` | Python не установлен или не добавлен в PATH — см. Шаг 1 |
| `Обученные модели не найдены` | Папка `models/` пустая — либо она не скачалась при клонировании, либо нужно её распаковать/докачать вручную |
| Порт 8501 занят | `streamlit run webapp/app.py --server.port=8502` и открыть `localhost:8502` |
| Работаете в Dev Container / Codespaces | Порт пробросится автоматически — откройте вкладку **PORTS** внизу VS Code и кликните на порт 8501 |

## Что внутри

- **Датасет**: [TrashNet](https://huggingface.co/datasets/garythung/trashnet), 6 классов, 2527 фото.
- **5 обученных архитектур** (transfer learning на ImageNet-претрейне, дообучены на CPU):
  MobileNetV2, MobileNetV3-Small, ResNet18, ShuffleNetV2 x1.0, EfficientNet-B0.
- **Сравнение по качеству и скорости** — таблица метрик, confusion matrix для
  каждой модели, анализ типичных ошибок (`reports/ANALYSIS.md`).
- **Веб-интерфейс на Streamlit** — загрузка фото, выбор модели, вывод класса,
  вероятности по всем классам и рекомендации по сортировке.

Лучшая модель по итогам сравнения — **EfficientNet-B0** (macro F1 = 0.868,
~11 мс/фото на CPU, 15.6 МБ). Подробности и разбор путаницы похожих
материалов (glass↔plastic, cardboard→paper, металл как "ловушка" для
блестящих поверхностей) — в [`reports/ANALYSIS.md`](reports/ANALYSIS.md).

## Структура проекта

```
data/           данные (raw/split — генерируются, в git не попадают)
src/
  prepare_data.py   разбиение датасета на train/val/test
  dataset.py        загрузка данных, аугментации
  models.py         фабрика 5 архитектур (torchvision)
  train.py          обучение (head warmup + fine-tuning)
  evaluate.py        метрики, confusion matrix, замер скорости на тесте
  compare_models.py  сравнение моделей, выбор лучшей, анализ путаницы
  inference.py       общая логика инференса (использует и evaluate, и webapp)
  waste_classes.py   классы отходов, контейнеры, рекомендации
models/          обученные веса (*.pt) для всех 5 архитектур — уже в репозитории,
                 переобучать не нужно, чтобы просто протестировать интерфейс
reports/         метрики, confusion matrix (png), сравнительная таблица, ANALYSIS.md
webapp/app.py    Streamlit-приложение
scripts/         run_pipeline.sh (полный пайплайн), run_webapp.sh (запуск UI)
```

## Полный пайплайн с нуля (данные → обучение → оценка → сравнение)

Нужно, только если хотите переобучить модели заново (например, на своих
данных). Для обычного тестирования веб-интерфейса это делать не нужно —
готовые модели уже лежат в `models/`.

```bash
./scripts/run_pipeline.sh
```

Или пошагово из `src/`:

```bash
python3 prepare_data.py                                   # train/val/test split
python3 train.py --arch all --epochs-head 6 --epochs-finetune 6  # ~20 мин на 4 CPU
python3 evaluate.py --arch all                             # метрики + confusion matrix
python3 compare_models.py                                  # таблица сравнения + выбор лучшей
```

Обучить/оценить одну архитектуру: `python3 train.py --arch resnet18`.
